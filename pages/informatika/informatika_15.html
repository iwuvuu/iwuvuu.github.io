Editory

Editor určuje způsob jakým budete muset webové stránky vytvářet. Liší se v uživatelském rozhraní (jejich jednoduchost, intuitivnost a potřeba znalostí uživatele) nebo v jejich funkcionalitě (možnost náhledu výsledku, automatické doplňování kódu, validace kódu).

WYSIWYG

= “What You See Is What You Get”

WYSIWYG editory jsou vizální editory webových stránek určené pro všechny. Jejich největší výhodou je možnost vytvářet chtěnou webovou stránku bez nutnosti znalosti psát kód. Fungují na principu grafických editorů (Photoshop, GIMP…) nebo textových procesorů (Microsoft Word, LibreOffice Writer…), tudíž nám prezentují nabídku funkci, které nám pouze stačí vybrat a aplikovat. Daný výběr je poté přeložen pomocí šablony do vygenerovaného kódu.

Největší výhodou je samozřejmě absence kódování. Editory jsou tedy jednoduché pro začátečníky, kteří chtějí rychle vytvořit svoji webovou stránku. Velkou výhodou je také možnost náhledu výsledné stránky v reálném čase.

Nevýhodou je pak menší flexibilita a méně možností, jelikož je možné použít prvky dostupné v daném editoru. Automatická generace kódu může být problematická. Pokud bycom cthěli vytvořit vlastní funkci, která není dostupná v editoru může kód být nepřehledný a budeme muset ho prvně zanalyzovat. K´d také může být neefektivní, což v dnešní době moc lidí neřeší (chyba).

Příkladem WYSIWYG editorů můžou být: - Adobe Dreamweaver - Wordpress - Wix - Google Sites

Textové editory

Textové editory umožňují přímé psaní kódu. Je nutné znát strukturu webových stránek, HTML, CSS, JS… a být schopný svoji stránku v těchto jazycích sestavit. Vlastnosti jednotlivých editorů se velmi liší od editoru k editoru. Existují jednoduché nespecifické editory jako je notepad++; vim, které poskytují základní funkce programovacích editorů. Nebo robustnější IDE jako je příkladně VisualCode Studio, které již ze základu přináší mnoho funkcí zlepšující práci. Ale existují také specifické editory, které jsou přímo určené na psaní html kódu. Tyto editory už mohou mít funkce jako jsou náhled výsledné stránky nebo automatické doplňování html bloků.

Hlavní výhdou je maximální kontrola nad kódem a jeho upravení dle preference a potřeb. Větší potenciál v implementaci různých funkcí do stránky.

Nevýhodami pak jsou nutná znalost vývoje stránek a časová náročnost se kterou se takový vývoj mmusí potýkat. I se znalostmi může být kód a pokročilá funkcionalita neintuitivní při tvoření.

Static site generator

Takovým mezikrokem mezi WYSIWYG editory a textovými editory jsou generátory statických stránek. Pracují podobně jako WYSIWYG editory s šablonami. Princip spočívá v tom, že uživatel napíše obsah stránky v jednoduším jazyce nebo jiné možnosti zápisu dat (Markdown, JSON…) a zvolená šablona potom přeloží obsah do html kódu. Velkou vlastností šablon je jejich tvoření, které lze vytvořit s jistou dávkou znalostí, jakmile máme vytvořenou vlastní šablonu, můžeme potom jen přidávat další stránky pomocí jenoduchých příkazů. Nejlépe se SSG dají využít k různým blogům a méně náročným stránkám.

Mým oblíbeným příkladem generátoru statických stránek je HUGO.

Struktura webových stránek

HTML

=“Hyper Text Markup Language” Jedná se o základ webových stránek. Skládá se z elementů a každý element musí mít začáteční tag a ukončující tag. Elementy říkají prohlížeči jak zobrazit daný obsah

Příklad kódu psaném v HTML:

<!DOCTYPE html>
<html>
    <head>
        <title>Název stránky</title>
    </head>
    <body>
        <h1>Toto je nadpis s váhou 1 (největší)</h1>
        <h2>Toto je nadpis s váhou 2</h2>
        <p>Toto je paragraf textu</p>

        <a href="https://www.super-stránka.com">Toto je odkaz</a>

        <img src="https://super-stránka.com/obrázek.png"
            alt="Aletrnativní popis obrázku">
        </img>

        <div>
            <ul>
                <li>Toto</li>
                <li>Je</li>
                <li>Seznam</li>
            </ul>
            <p> Tento odstavec je společně s seznamem nahoře součástí tohoto div elementu</p>
        </div>
    </body>
</html>

Základní tagy

  • <!DOCTYPE html> - deklaruje, že se jedná o HTML5 dokument
  • <html> - základní element webové stránky
  • <head> - element obsahující metainformace o webové stránce
  • <title> - název stránky
  • <body> - element obsahující tělo webové stránky -> všechen její viditelný obsah
  • <h1> - Nadpis úrovně 1
  • <p> - odstavec textu
  • <div> - element určující blok, váže jiné elementy do jednoho celku

CSS

Stylistická část webové stránky. Nastavuje nám všchny barvy odsazení, fonty a velikosti fontů atd. Lze jej psát přímo do html kódu:

<style>
    body {
        background-color: lightblue;
    }
    h1 {
        color: white;
        text-align: center;
    }
    p {
        font-family: verdana;
        font-size: 20px;
    }
</style>

Nebo do samostatného .css souboru. Abychom tento soubor aplikovali, tak ho musíme odkázat v HTML souboru:

<head>
    <link rel="stylesheet" href="mystyle.css">
</head>

A potom už můžeme pokračovat jakobychom psali přímo do <style> elementu.

JavaScript

Javascript přidává možnost programování do jisté míry a stojí za pokročilými efekty a chováním stránky. Je to jediný programovací jazyk na webu.

Validace webových stránek

Validace slouží k zjistšění chyb a nesrovnalostí v kódu nebo třeba k zjištění špatných a rozbitých odkazů. Validace zajistí kompatibilitu napříš prohlížeči a zlepší tak přístupnost.

Validovat můžeme online přes různé webové stránky. Např. W3C Validation Service (validator.w3.org).

Umístění webu na server

Kroky k umístšní webu na server:

  1. Vybrat hostingovou službu. To může být cloudový server, kde si platíme hosting. Nebo to může být náž server doma (klidně i starý nepotřebný počítač)
  2. Zaregistrovat doménu (webová adresa, kterou budou uživatelé psát do vyhledávače www.vase-stranka.com). Vyhledáme si volnou doménu, kterou chceme a koupíme si ji. A propojit ji s naším serverem
  3. Nahrát soubory na server.
  4. Ujistit se, že všechny odkazy a soubory jsou správně propojeny
  5. Zajistit všechny potřebné ochrané vrstvy.
